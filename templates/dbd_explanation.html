Apa itu Backward Chaining?

Backward chaining adalah metode inferensi yang bekerja mundur dari goal (tujuan) menuju fakta-fakta yang mendukung. Sistem dimulai dengan hipotesis "Apa tingkat risiko DBD?" lalu mencari aturan yang dapat membuktikan hipotesis tersebut.

📊 Alur Backward Chaining pada Sistem DBD
                    ┌─────────────────────────────────────────────────────────────┐
                    │              GOAL: tingkat_resiko_dbd = ?                   │
                    │                     (RENDAH / SEDANG / TINGGI)              │
                    └─────────────────────────┬───────────────────────────────────┘
                                              │
                    ┌─────────────────────────▼───────────────────────────────────┐
                    │         Cari aturan yang memiliki konsekuen = goal          │
                    │                  Rules 1-13 (Set 1)                          │
                    └─────────────────────────┬───────────────────────────────────┘
                                              │
                    ┌─────────────────────────▼───────────────────────────────────┐
                    │   Antecedents aturan: potensi_perkembangbiakan, iklim,      │
                    │                    faktor_eksposur_manusia                   │
                    └─────────────────────────┬───────────────────────────────────┘
                                              │
                    ┌─────────────────────────▼───────────────────────────────────┐
                    │         Sub-goal: Buktikan potensi_perkembangbiakan         │
                    │                 Cari Rules 14-39 (Set 2)                     │
                    └─────────────────────────┬───────────────────────────────────┘
                                              │
                    ┌─────────────────────────▼───────────────────────────────────┐
                    │         Sub-goal: Buktikan iklim (Rules 40-44 Set 3)        │
                    └─────────────────────────┬───────────────────────────────────┘
                                              │
                    ┌─────────────────────────▼───────────────────────────────────┐
                    │      Sub-goal: Buktikan faktor_eksposur (Rules 45-49 Set 4) │
                    └─────────────────────────┬───────────────────────────────────┘
                                              │
                    ┌─────────────────────────▼───────────────────────────────────┐
                    │              Semua terpenuhi → FIRE RULE                     │
                    │                 Goal berhasil dibuktikan                     │
                    └─────────────────────────────────────────────────────────────┘

📋 Struktur Knowledge Base (4 Set Aturan)

Set 1

Rules 1-13

Tingkat Risiko DBD

Set 2

Rules 14-39

Potensi Perkembangbiakan

Set 3

Rules 40-44

Kondisi Iklim

Set 4

Rules 45-49

Faktor Eksposur Manusia

🔍 Contoh Backward Chaining

Langkah 1: Sistem ingin membuktikan tingkat_resiko_dbd = tinggi
Langkah 2: Cari aturan dengan konsekuen tersebut → Rule 4, 9, 10
Langkah 3: Coba Rule 9: IF potensi=tinggi AND iklim=mendukung THEN risiko=tinggi
Langkah 4: Butuh bukti potensi=tinggi → cari Rules 19, 21, 25, 26
Langkah 5: Butuh bukti iklim=mendukung → cari Rules 40, 43
Langkah 6: Semua terbukti → Rule 9 terpicu → Goal terbukti

✅ Kelebihan Backward Chaining untuk Kasus DBD

  • Goal-oriented: Fokus hanya pada fakta yang relevan dengan risiko DBD
  • Efisien: Tidak perlu mengevaluasi semua aturan yang tidak relevan
  • Explanation facility: Mudah menjelaskan mengapa suatu kesimpulan diambil
  • Hierarchical: Cocok dengan struktur knowledge base yang berlapis (Set 1 → Set 2,3,4)
💡 Perbedaan dengan Forward Chaining:
Forward Chaining: Fakta → Kesimpulan (data-driven)
Backward Chaining: Hipotesis → Fakta pendukung (goal-driven)
Sistem ini menggunakan backward chaining karena kita ingin membuktikan tingkat risiko DBD dari data yang tersedia.
